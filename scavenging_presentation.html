// 01. PROBLEM DEFINITION

The State of Stale Records

Cloud Drift

Cloud instances are ephemeral, but DNS records are persistent. Destroying a VPC or VM via Terraform/Ansible often leaves orphaned A and CNAME records pointing to dead space.

Temporal Mismatch

Microsoft DNS relies on Aging/Scavenging intervals (temporal), which fails to account for actual network behavior (queries). This leads to "Zombie" records that look active but have zero queries.

IPAM Fragmentation

Orphaned PTR records prevent IP reuse. In high-churn environments like AWS, this leads to artificial "IP Starvation" where subnets appear full but are actually empty.

// 03. PROPOSED SOLUTIONS

Comparison for Discussion

Option A: Native CNA

Low Effort

Leverage Infoblox Cloud Network Automation (CNA). Native API polling between the Grid and AWS/Azure.

  • ✔ Managed by Infoblox Support
  • ✔ Native "Reclaimable" state
  • ✖ Requires specialized licensing
  • ✖ Limited to CNA-supported regions

Option B: Async WAPI CLI

Custom Flex

Build a specialized Python microservice using Async WAPI to bridge Cloud/On-Prem with a single "Reaper" logic.

  • ✔ Highly customizable behavioral rules
  • ✔ Supports non-CNA cloud providers
  • ✖ Requires internal dev maintenance
  • ✖ High initial implementation labor

Option C: EA Governance

Strict Tagging

Manual policies enforced via Extensible Attributes (EAs). Records are deleted only when a "Lifecycle" tag expires.

  • ✔ Most secure (No auto-delete)
  • ✔ Zero licensing impact
  • ✖ High manual review overhead
  • ✖ Relies on 100% accurate tagging
